- **Smell:** `vacuous-assertion`
- **Rationale:** The test claims no-op behavior for an empty SVG map but only asserts `not_to raise_error`; this matches the weak-oracle signal where a side-effectful wrong implementation (creating directories, touching files, or writing placeholders) can still pass because no side-effect channel is asserted. See [vacuous-assertion](https://texarkanine.github.io/slobac/taxonomy/vacuous-assertion/).
- **Prescribed remediation:** Keep the no-exception assertion as secondary and add primary assertions on observable no-op outcomes (for example: destination output directory remains absent, no new files created, and no copy invocations happen).
- **Why this isn't a false positive:** This is not a legitimate "negative-only contract fully verified" case because the test names "does nothing" yet never checks absence of side effects.
